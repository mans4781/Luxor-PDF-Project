## luxor-reader-desktop — "Luxor PDF Reader" (`com.luxor.pdfreader`)
- Wraps `luxor-pdf`. Hybrid loading via `LUXOR_LOAD_MODE`: `auto` (default) loads `https://luxorpdf.com/luxor-pdf/`, falls back to bundled offline copy (`app://luxor/`, SPA fallback + traversal guard) on `did-fail-load`; `remote`/`bundled` force a mode.
- Icon: `build/icon.ico` (16–256) generated by `run icon:regen` from `build/icon-source.png` (user-supplied LUXOR shield, bg removed + trimmed, checked in). One .ico covers electron-builder win/nsis icons and web stub MUI_ICON.
- `electron-updater` auto-updates from GitHub Releases — publish config `mans4781/Luxor-PDF-Project` (the repo the Replit workspace Git pane is connected to). First release v0.1.0 published 2026-07-09 (full installer 79.6 MB + latest.yml).
- `scripts/build-web-bundle.mjs` builds luxor-pdf with `BASE_PATH=./` into `web-bundle/`.
- CI `.github/workflows/reader-desktop-windows.yml` (windows-latest): manual dispatch or `reader-v*` tags; publish needs `GH_RELEASES_TOKEN` repo secret. See memory `windows-ci-pnpm.md` for the required workspace-override stripping.
- **Web-installer stub** (`web-installer/web-setup.nsi` + `download-full.ps1`; `run dist:web-stub` → `dist/Luxor PDF Reader Web Setup.exe`, ~200 KB): reads `latest.yml` from GitHub Releases `latest/download`, downloads full installer to `$PLUGINSDIR` (sha512-verified), runs it. Compiles on Linux (`makensis` from nixpkgs); CI installs NSIS via choco. Unsigned → SmartScreen warning until code-signed.
- **Default PDF reader**: electron-builder `fileAssociations` registers `.pdf` (role Viewer). Single-instance main; double-clicked PDF path arrives via `process.argv` (stored `pendingPdfPath`) or `second-instance` argv, read in main (512 MB cap), handed to renderer via `luxor:get-pending-file` (one-shot invoke, sets `rendererReady`) / `luxor:open-file` (push). `isTrustedIpcSender()` origin check on both handlers. Preload exposes `getPendingFile()`/`onOpenFile()`; luxor-pdf `src/lib/desktopBridge.ts` (`initDesktopFileOpen`, wired in App.tsx) converts bytes to `File` → `setFile`.

## luxor-desktop — "Luxor PDF Secure" (`com.luxor.pdfsecure`)
- Wraps `pdf-expiry`; Windows-only NSIS (`oneClick=false`, choose dir, Desktop+Start shortcuts, `deleteAppDataOnUninstall=false` so reinstall keeps device id/license slot).
- Load modes: `LUXOR_LOAD_MODE=remote` (default `https://luxorpdf.com/pdf-expiry/`) or `bundled` (built pdf-expiry via `extraResources`).
- Hardened window: `contextIsolation`, no `nodeIntegration`, `sandbox=true`. Preload exposes only `window.luxor = { isDesktop, getDeviceId(), getAppInfo() }`. External links → system browser; cross-origin nav blocked in remote mode.
- Device id: UUID v4 at `<userData>/device-id.txt`. `pdf-expiry/src/main.tsx` `initDeviceIdFromBridge()` mirrors it into `localStorage["luxor.deviceId"]`; `setDeviceIdGetter(...)` from `@workspace/api-client-react` adds `X-Device-Id` header to every API request.
- Scripts: `run start` (dev window), `run dist:win` (needs Windows/Wine — not runnable in this Linux container), `run icon:regen`.
