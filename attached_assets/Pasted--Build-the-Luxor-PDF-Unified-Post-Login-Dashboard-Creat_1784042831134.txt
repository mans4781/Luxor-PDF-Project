# 16. Coming-Soon Behaviour Across the Dashboard

Apply these rules everywhere:

* eSign must always display a `Coming Soon` status.
* eSign must not show `Full Access`.
* eSign must not show an active product entitlement.
* eSign must not have an `Open App` button.
* eSign must not appear in recent product activity.
* eSign must not appear as an available quick action.
* eSign routes must not expose unfinished functionality.
* eSign may be shown in navigation, search and product switching for awareness.
* Clicking eSign should open a coming-soon modal or informational page.
* Users may join a waitlist or request launch notifications.
* The system should store waitlist consent and submission date.
* Prevent duplicate waitlist submissions for the same email.
* Show a confirmation when the user is already registered.

Use product data such as:

```ts
type ProductAccess = {
  id: "reader" | "secure" | "esign";
  name: string;
  status:
    | "full_access"
    | "trial"
    | "upgrade_required"
```

Example:

```ts
const products: ProductAccess[] = [

    name: "Luxor PDF Reader",
```
