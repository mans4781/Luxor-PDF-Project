In my Luxor PDF Reader, the Highlight tool is not working like Adobe Acrobat or Microsoft Edge PDF Reader.

I want to redesign/fix the Highlight tool behavior so it feels professional and natural.

Goal:
Implement Adobe/Microsoft Edge-style text selection and highlighting.

Main behavior I want:
1. When the Highlight/Text Selection tool is active, the cursor should become a vertical text-selection cursor, like an I-beam cursor.
2. When the user moves/drags the cursor horizontally over text, the PDF text should be automatically selected.
3. User should be able to drag across words, lines, and paragraphs naturally.
4. When the user stops selecting and right-clicks on the selected text, a small popup/context menu should appear.
5. The popup should allow the user to choose:
   - Highlight color
   - Underline
   - Strike-through if possible
   - Copy text
   - Delete annotation if selected annotation is clicked
6. After selecting a highlight color, only the selected text should be highlighted.
7. The selection and highlight should look smooth and uniform like Adobe Acrobat / Microsoft Edge.

Current problem:
- Highlight tool does not behave like Adobe or Microsoft Edge.
- Text selection is not natural.
- Highlight is not accurate.
- There is no proper I-beam text cursor.
- There is no right-click popup for highlight color / underline options.
- Text selection/highlight appears rough and not professional.

Required Highlight Tool Behavior:

1. Cursor behavior:
- When highlight/text selection mode is active, show a vertical I-beam cursor over PDF text.
- Cursor should indicate that text can be selected.
- Over non-text areas, cursor can remain default or crosshair.
- The user should be able to select text by dragging horizontally or vertically across text.

2. Text selection behavior:
- Use the PDF.js text layer for real text selection.
- Do not create fake selection manually unless needed.
- Allow normal browser/PDF text selection behavior.
- When user drags from left to right or right to left, text should auto-select naturally.
- Selection should work for:
  - one word
  - one sentence
  - one full line
  - multiple lines
  - paragraph
  - two-column PDF layout
- The selection should not jump or select unrelated text.
- The selection overlay should be uniform:
  - no gaps between words
  - one smooth strip per selected line
  - no giant rectangle over the whole paragraph
  - no merging across columns

3. Right-click popup behavior:
- When the user selects text and right-clicks, show a custom popup near the selected text or cursor.
- The popup should include:
  - Yellow highlight
  - Green highlight
  - Blue highlight
  - Pink highlight
  - Underline
  - Strike-through if possible
  - Copy
  - Cancel / close
- The popup should look modern, small, and similar to Adobe/Microsoft Edge style.
- The popup should not block the selected text too much.
- Clicking outside the popup should close it.
- Pressing Esc should close it.
- Popup should only appear if there is selected PDF text.
- If no text is selected, do not show the highlight popup.

4. Highlight application:
- When user selects a color in the popup, create a highlight annotation from the selected text.
- Highlight must cover only selected text.
- Spaces between words should be filled.
- Multi-line selection should create separate line rectangles.
- Highlight must be saved as annotation data.
- Highlight should remain correctly positioned after:
  - zoom
  - scroll
  - page change
  - reload if annotation saving exists

5. Underline behavior:
- When user selects Underline from popup:
  - Add an underline annotation under the selected text.
  - Underline should follow each selected line separately.
  - Store it as annotation type: "underline".
  - It should remain aligned after zoom and scroll.
- Underline should not cover the text.
- Underline should be removable by eraser.

6. Strike-through behavior, if possible:
- When user selects Strike-through:
  - Add a line through the middle of selected text.
  - Store it as annotation type: "strike".
  - It should align properly with selected text.
- If strike-through is difficult, implement underline first and leave strike-through as optional.

7. Eraser compatibility:
- Eraser should remove the touched annotation only.
- If eraser touches a highlight annotation, remove that highlight.
- If eraser touches an underline annotation, remove that underline.
- If eraser touches strike-through annotation, remove that strike annotation.
- Do not erase original PDF text.
- Do not erase all annotations.
