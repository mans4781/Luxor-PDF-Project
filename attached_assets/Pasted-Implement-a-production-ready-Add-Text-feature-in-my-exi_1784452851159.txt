Implement a production-ready “Add Text” feature in my existing Luxor PDF desktop/web application.

The feature’s technical name is “Free Text Annotation” or “Typewriter Tool.” It must allow users to click anywhere on a PDF page and insert an editable text box, similar to Adobe Acrobat’s Add Text tool.

IMPORTANT:
- Work inside my existing project.
- Do not redesign or break the current PDF viewer.
- Preserve the existing Luxor PDF layout, navigation, zoom, thumbnails, toolbar and colour theme.
- Use the application’s existing design variables and components.
- Use TypeScript and clean, modular code.
- Do not create only a visual demo. Build the complete working feature.

TECHNOLOGY:
- React + TypeScript
- Electron-compatible implementation
- PDF.js for rendering PDF pages
- pdf-lib or another suitable open-source PDF library for saving text permanently into the PDF
- Use an HTML/SVG annotation overlay positioned above each PDF.js page
- Do not convert or rasterize the complete PDF

==================================================
1. TOOLBAR BUTTON
==================================================

Add a toolbar button with:

Icon: “T” inside a text-box style icon
Label: “Add Text”
Tooltip: “Add text anywhere on the PDF”
Keyboard shortcut: T

When the user clicks “Add Text”:
- Activate text insertion mode
- Highlight the toolbar button
- Change the cursor to a text cursor or crosshair
- Display a small instruction: “Click anywhere on the page to add text”
- Pressing Escape must exit Add Text mode

==================================================
2. CREATE A TEXT BOX
==================================================

When the user clicks anywhere inside a PDF page:

- Create a transparent editable text box at the exact clicked position
- Automatically place the typing cursor inside it
- Allow the user to type immediately
- Default placeholder must not be saved
- The text box must expand vertically while typing
- Default width: approximately 220 px
- Minimum width: 60 px
- Minimum height: 28 px
- Initial text must be aligned to the top-left
- Support multiple text boxes on every PDF page

Default formatting:
- Font: Helvetica or Arial
- Font size: 14 pt
- Text colour: black
- Transparent background
- No visible border after deselection
- Line height: approximately 1.3

==================================================
3. SELECT, MOVE AND RESIZE
==================================================

When a text box is selected:

- Show a subtle bounding box
- Show resize handles on corners and edges
- Allow dragging the text box anywhere on the same PDF page
- Allow resizing horizontally and vertically
- Keep the text inside the resized area
- Do not allow the box to move outside the PDF page
- Use pointer events so dragging works smoothly
- Do not trigger page scrolling while dragging

Selection behaviour:
- Single click selects the text box
- Double-click enters text-editing mode
- Clicking outside deselects it
- Delete or Backspace removes the selected text box when not editing text
- Escape exits editing or deselects the object
- Ctrl/Cmd + C copies the selected text box
- Ctrl/Cmd + V pastes a duplicate with a slight offset
- Ctrl/Cmd + D duplicates it
- Arrow keys move it by 1 px
- Shift + Arrow keys move it by 10 px

==================================================

Display a floating formatting toolbar when a text box is selected.

Include:

- Font family dropdown

- Increase font size
- Decrease font size

- Underline
- Text colour picker
- Background colour picker
- Background transparency option

- Delete

Supported fonts:
- Helvetica
- Times New Roman
- Courier
- Arial
- Georgia

If a selected font cannot be embedded safely, use a reliable fallback font.

Formatting changes must apply:
- To selected text when the user highlights only part of the content, where technically supported
- Otherwise, to the entire selected text box

5. TEXT EDITING EXPERIENCE
==================================================

Use a contenteditable element, textarea, or another stable editing approach.

Requirements:
- Proper text cursor positioning
- Multiline text
- Enter creates a new line
- Text wrapping
- Copy and paste text
- Undo and redo
- No unexpected line breaks
- No text blur during zoom
- Prevent browser spellcheck marks from becoming part of the saved PDF

- Support common English characters, numbers and punctuation
- Structure the code so Unicode font embedding can be added later

Do not allow editing shortcuts to accidentally trigger viewer shortcuts while typing.

==================================================
6. PDF ZOOM AND COORDINATE HANDLING
==================================================

This is critical.

The text box must remain attached to the correct PDF coordinates when:
- Zooming in
- Zooming out
- Changing fit-to-width mode
- Changing fit-to-page mode
- Resizing the application window
- Scrolling between pages
- Rotating a page

Store annotation positions using PDF page coordinates, not only screen pixels.

Create reliable conversion utilities for:
- Screen coordinates to PDF coordinates
- PDF coordinates to screen coordinates
- PDF’s bottom-left coordinate system
- Browser’s top-left coordinate system
- Current page scale
- Page rotation
- Device pixel ratio

After changing zoom, the text must remain in exactly the same document position.

9. SAVE OPTIONS
==================================================

Provide two save behaviours:

A. Save as Editable Annotation
- Preserve text-box metadata so it can be edited again when reopened inside Luxor PDF
- Store annotation metadata safely
- Where feasible, create a standards-compliant PDF /FreeText annotation
- Use PDF annotation subtype /FreeText
- Keep the PDF valid and compatible with common PDF readers

B. Flatten Text into PDF
- Draw the text permanently onto the PDF page
- Preserve the original PDF quality
- Do not rasterize pages
- Embed the required font
- Correctly draw multiline text
- Respect wrapping, font size, colour, opacity, alignment and rotation
- The flattened text must appear in Adobe Acrobat, browsers and other PDF readers
- Remove the editable overlay after successful flattening
- Save as a new PDF rather than overwriting the original without confirmation

Add save options:

- “Save PDF”
- “Save As”
- “Flatten Annotations”
- “Keep annotations editable”

If standards-compliant editable /FreeText annotations are difficult with the chosen library, fully implement the reliable flattened save first, while preserving editable annotation data within the Luxor project/session. Do not fake PDF saving.
