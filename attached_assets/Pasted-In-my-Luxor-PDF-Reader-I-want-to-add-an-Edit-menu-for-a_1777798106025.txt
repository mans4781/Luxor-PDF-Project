In my Luxor PDF Reader, I want to add an “Edit” menu for an opened PDF.

Please add the following features under the Edit menu:

1. Redact
2. Add Image
3. Add Watermark
4. Add Page No.
5. Edit Text
6. Compress This PDF

Goal:
Create a clean, professional Edit menu similar to modern PDF software like Adobe Acrobat, Foxit, and Microsoft Edge style, but matching Luxor PDF Reader branding.

-----------------------------------
EDIT MENU STRUCTURE
-----------------------------------

Add this menu in the top toolbar/header:

Edit
├── Redact
├── Add Image
├── Add Watermark
├── Add Page No.
├── Edit Text
└── Compress This PDF

Each option should have:
- clean icon
- label
- hover state
- active/selected state if applicable
- tooltip if needed

-----------------------------------
1. REDACT
-----------------------------------

Feature:
Allow the user to permanently hide/remove sensitive text or areas from the PDF.

Required behavior:
- User clicks Edit > Redact.
- Cursor changes to crosshair or selection cursor.
- User can select text or draw a rectangle over the area to redact.
- Selected area should show a redaction preview box.
- Default redaction color: black.
- Provide an “Apply Redaction” button.
- Before final apply, show warning:
  “Redaction permanently removes/obscures selected content. This action cannot be reversed after saving.”
- Once applied, the selected area should be permanently covered/removed in the exported/saved PDF.
- Redaction should not be only a visual annotation if the PDF is saved.
- It must actually burn into the PDF output.

Optional:
- Add redaction colors:
  - Black
  - White
- Add label text option:
  - “REDACTED”

Important:
- Do not only draw a black rectangle as a temporary annotation.
- When user saves/export PDF, redacted area must be flattened/permanent.

-----------------------------------
2. ADD IMAGE
-----------------------------------

Feature:
Allow user to insert an image into the open PDF.

Required behavior:
- User clicks Edit > Add Image.
- Open file picker.
- Allow PNG, JPG, JPEG, WEBP.
- After selecting image, show it on the PDF page.
- User can drag to position the image.
- User can resize using corner handles.
- User can rotate if possible.
- User can delete selected image.
- Image position should stay correct after zoom/scroll.
- Save/export should embed the image into the PDF.

Image annotation/object data should include:
- id
- pageNumber
- image source/blob/base64
- x
- y
- width
- height
- rotation
- opacity
- createdAt

-----------------------------------
3. ADD WATERMARK
-----------------------------------

Feature:
Allow user to add text or image watermark to PDF.

Required behavior:
- User clicks Edit > Add Watermark.
- Open watermark settings modal/panel.
- User can choose:
  - Text watermark
  - Image watermark
- Text watermark options:
  - watermark text
  - font size
  - font family
  - color
  - opacity
  - rotation angle
  - position
- Image watermark options:
  - upload image
  - opacity
  - size
  - rotation angle
  - position
- Page range options:
  - all pages
  - current page
  - custom page range
- Common positions:
  - center
  - top-left
  - top-right
  - bottom-left
  - bottom-right
  - tiled/repeated watermark
- Default watermark:
  - text: “CONFIDENTIAL”
  - opacity: 0.18
  - rotation: -35 degrees
  - position: center
- Preview watermark before applying.
- Save/export should apply watermark to the PDF.

-----------------------------------
4. ADD PAGE NO.
-----------------------------------

Feature:
Allow user to add page numbers to PDF pages.

Required behavior:
- User clicks Edit > Add Page No.
- Open page number settings modal/panel.
- User can choose:
  - position: bottom center, bottom right, bottom left, top center, top right, top left
  - start number
  - page range
  - font size
  - font family
  - text color
  - prefix/suffix
- Example:
  - Page 1
  - 1
  - 1 of 10
  - Page 1 of 10
- Default:
  - position: bottom center
  - font size: 11
  - color: #333333
  - format: “Page {page} of {total}”
- Show preview before applying.
- Save/export should embed page numbers into the PDF.

-----------------------------------
5. EDIT TEXT
-----------------------------------

Feature:
Allow user to edit existing PDF text where possible.

Required behavior:
- User clicks Edit > Edit Text.
- Detect editable text layer or PDF text objects if supported.
- User can click existing text and edit it.
- Show editable text box overlay aligned with selected text.
- Preserve font size, approximate font family, color, and position as much as possible.
- If exact PDF text editing is not possible with current PDF library, implement safe overlay-based text editing:
  - cover old text with white rectangle
  - place new editable text on top
  - save/export as flattened PDF
- User can:
  - change text
  - change font size
  - change color
  - delete selected text object
- Add warning if using overlay method:
  “This edit replaces the visible text using an overlay and will be flattened when saved.”

Important:
- Do not break text selection, highlight, draw, eraser, or add text tools.
- Keep text edits aligned after zoom and scroll.
- Save/export must preserve visible changes.

-----------------------------------
6. COMPRESS THIS PDF
-----------------------------------

Feature:
Allow user to compress the currently opened PDF.

Required behavior:
- User clicks Edit > Compress This PDF.
- Open compression modal/panel.
- Show options:
  - Low compression / Best quality
  - Medium compression / Balanced
  - High compression / Smaller file
  - Custom compression if possible
- Show original file size.
- After compression, show compressed file size.
- Show percentage reduction.
- Provide buttons:
  - Compress
  - Download Compressed PDF
  - Replace Current PDF if safe
- Compression should reduce:
  - image quality/resolution where possible
  - unnecessary metadata
  - unused objects if supported
- Do not corrupt the PDF.
- Keep text readable.
- Keep pages in correct order.
